# Accessibility Testing Web

Use this skill as a basic starting point.

## Workflow

1. Review the current UI flows and accessibility requirements.
2. Identify likely issues in semantics, focus, labels, and contrast.
3. Check interactive flows with keyboard and screen reader expectations in mind.
4. Keep fixes aligned with existing UI patterns.
5. Verify improvements across common user journeys.

## Output

- Produce practical accessibility focused improvements.
- Keep fixes user centered and maintainable.
- Prefer semantic corrections over visual-only adjustments.
