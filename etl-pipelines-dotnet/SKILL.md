# ETL Pipelines .NET

Use this skill as a basic starting point.

## Workflow

1. Review the source, transform, and target data boundaries.
2. Identify schema mapping, validation, and scheduling requirements.
3. Keep transformation steps explicit and traceable.
4. Handle bad records and partial failures intentionally.
5. Verify pipeline behavior with representative data.

## Output

- Produce practical ETL workflow changes.
- Keep data movement predictable and maintainable.
- Prefer explicit transformation stages over monolithic processing.
