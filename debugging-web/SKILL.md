# Debugging Web

Use this skill as a basic starting point.

## Workflow

1. Review the reported issue, reproduction path, and current code flow.
2. Identify whether the problem is UI, network, state, or environment related.
3. Narrow the issue before attempting broad fixes.
4. Keep changes targeted to the verified root cause.
5. Verify the original issue is resolved without regressions.

## Output

- Produce practical debugging oriented fixes.
- Keep investigation structured and evidence driven.
- Prefer root cause fixes over symptom patches.
