# Repo Documentation

Use this skill as a basic starting point.

## Workflow

1. Review the repository purpose, setup steps, and user audience.
2. Identify missing onboarding, usage, or maintenance guidance.
3. Keep documentation concise and structured.
4. Match wording to the technical maturity of the project.
5. Verify instructions are actionable and accurate.

## Output

- Produce practical repository documentation improvements.
- Keep docs clear and easy to follow.
- Prefer concise, useful guidance over verbose filler.
