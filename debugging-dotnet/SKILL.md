# Debugging .NET

Use this skill as a basic starting point.

## Workflow

1. Review the failure symptoms, logs, and reproduction conditions.
2. Identify whether the issue is configuration, code, data, or environment related.
3. Narrow the failing path before applying changes.
4. Keep fixes targeted to the verified cause.
5. Verify the original behavior and likely adjacent cases.

## Output

- Produce practical debugging oriented fixes.
- Keep analysis evidence driven and focused.
- Prefer root cause resolution over superficial patches.
