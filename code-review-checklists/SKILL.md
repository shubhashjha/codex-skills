# Code Review Checklists

Use this skill as a basic starting point.

## Workflow

1. Review the type of code change and the review goals.
2. Identify correctness, security, maintainability, and testing concerns.
3. Keep checklist items concrete and applicable.
4. Avoid bloated lists that reviewers will ignore.
5. Verify the checklist matches the repository context.

## Output

- Produce practical review guidance.
- Keep checklists focused and actionable.
- Prefer concise review prompts over generic boilerplate.
