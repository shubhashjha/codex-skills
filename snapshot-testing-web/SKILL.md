# Snapshot Testing Web

Use this skill as a basic starting point.

## Workflow

1. Review current component tests and candidate snapshot targets.
2. Use snapshots only where output is stable and meaningful.
3. Keep snapshots small and easy to review.
4. Avoid using snapshots as a substitute for behavior assertions.
5. Verify snapshot updates are intentional.

## Output

- Produce practical snapshot testing changes.
- Keep snapshots readable and low-noise.
- Prefer focused regression checks over large brittle snapshots.
