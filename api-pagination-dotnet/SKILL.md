# API Pagination .NET

Use this skill as a basic starting point.

## Workflow

1. Review the current endpoint shape and dataset size.
2. Identify pagination mode, metadata, and filtering requirements.
3. Keep contracts explicit and consistent across endpoints.
4. Avoid unbounded list responses in production paths.
5. Verify query behavior and pagination edge cases.

## Output

- Produce practical paginated API changes.
- Keep list endpoints scalable and predictable.
- Prefer explicit pagination contracts over implicit defaults.
