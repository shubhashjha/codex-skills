# Penetration Testing Basics

Use this skill as a basic starting point.

## Workflow

1. Review the application surface and testing scope.
2. Identify common weak points such as auth, input handling, or sensitive endpoints.
3. Keep testing focused on defensive validation and safe analysis.
4. Document findings clearly and map them to mitigations.
5. Verify remediations where possible.

## Output

- Produce practical security review guidance and fixes.
- Keep findings actionable and bounded.
- Prefer responsible defensive analysis over broad speculation.
