# TanStack Router Web

Use this skill as a basic starting point.

## Workflow

1. Review the current route structure and navigation model.
2. Identify route typing, data loading, and nested layout needs.
3. Keep route definitions explicit and easy to scan.
4. Match navigation behavior with existing UI patterns.
5. Verify routing transitions and edge cases.

## Output

- Produce practical routing focused changes.
- Keep navigation maintainable and consistent.
- Prefer explicit route structure over hidden behavior.
