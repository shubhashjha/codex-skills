# Query Specifications .NET

Use this skill as a basic starting point.

## Workflow

1. Review the current query complexity and repetition.
2. Identify filters and query rules worth extracting into specifications.
3. Keep specifications focused and composable.
4. Avoid over-abstracting trivial query cases.
5. Verify resulting queries remain correct and readable.

## Output

- Produce practical query specification changes.
- Keep data access logic reusable and maintainable.
- Prefer focused query abstractions over generic indirection.
