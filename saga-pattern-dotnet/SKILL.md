# Saga Pattern .NET

Use this skill as a basic starting point.

## Workflow

1. Review the distributed workflow and failure recovery requirements.
2. Identify steps, state transitions, and compensating actions clearly.
3. Keep orchestration logic explicit and traceable.
4. Handle retries, idempotency, and partial failure paths intentionally.
5. Verify expected flows and compensation behavior.

## Output

- Produce practical saga workflow changes.
- Keep distributed coordination clear and maintainable.
- Prefer explicit state transitions over ad hoc flow logic.
