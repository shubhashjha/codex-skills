# Testing Library Web

Use this skill as a basic starting point.

## Workflow

1. Review the current component and test setup.
2. Identify meaningful user facing behavior to test.
3. Prefer accessible queries and realistic interaction flows.
4. Avoid overtesting implementation details.
5. Verify tests are stable and easy to read.

## Output

- Produce practical frontend test changes.
- Keep tests user focused and maintainable.
- Prefer behavior level assertions over internal details.
