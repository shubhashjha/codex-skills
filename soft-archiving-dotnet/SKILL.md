# Soft Archiving .NET

Use this skill as a basic starting point.

## Workflow

1. Review current delete and retention behavior.
2. Identify which records should be hidden, archived, or restored.
3. Keep archive flags and access rules explicit.
4. Avoid accidental inclusion of archived data in active workflows.
5. Verify create, archive, query, and restore scenarios.

## Output

- Produce practical archival behavior changes.
- Keep data lifecycle rules maintainable and safe.
- Prefer explicit archive handling over overloaded delete semantics.
