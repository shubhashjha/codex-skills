# Azure Monitoring

Use this skill as a basic starting point.

## Workflow

1. Review the application runtime and observability requirements.
2. Identify logs, metrics, alerts, and dashboards that matter operationally.
3. Keep monitoring high signal and tied to real failure modes.
4. Avoid excessive telemetry that is costly or low value.
5. Verify captured signals and alerting assumptions.

## Output

- Produce practical Azure monitoring improvements.
- Keep observability actionable and maintainable.
- Prefer focused operational visibility over noisy telemetry.
