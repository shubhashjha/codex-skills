# Data Seeding .NET

Use this skill as a basic starting point.

## Workflow

1. Review the current schema and startup initialization flow.
2. Identify which data is reference data versus sample data.
3. Keep seeding deterministic and environment aware.
4. Avoid destructive seeding logic in production paths.
5. Verify seeded data and repeat-run behavior.

## Output

- Produce practical data seeding changes.
- Keep initialization logic safe and predictable.
- Prefer idempotent seed behavior over one-time assumptions.
