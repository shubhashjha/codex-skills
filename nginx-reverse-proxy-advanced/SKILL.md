# Nginx Reverse Proxy Advanced

Use this skill as a basic starting point.

## Workflow

1. Review the current upstream services and traffic routing needs.
2. Identify TLS, headers, caching, compression, and rewrite requirements.
3. Keep configuration explicit and easy to audit.
4. Avoid risky rewrites without understanding request flow.
5. Verify proxy behavior with representative routes.

## Output

- Produce practical reverse proxy configuration changes.
- Keep request routing stable and maintainable.
- Prefer explicit server behavior over opaque configuration.
