# API Key Management .NET

Use this skill as a basic starting point.

## Workflow

1. Review the current authentication model and API key requirements.
2. Identify generation, storage, rotation, and revocation needs.
3. Keep validation logic explicit and secure.
4. Avoid exposing raw keys in logs or responses.
5. Verify authorized and unauthorized access paths.

## Output

- Produce practical API key management changes.
- Keep access control secure and reviewable.
- Prefer explicit lifecycle handling over simplistic key checks.
