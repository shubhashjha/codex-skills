# Swagger Node Web

Use this skill as a basic starting point.

## Workflow

1. Review the current route structure and API contracts.
2. Identify the documentation source of truth and gaps.
3. Keep schema definitions and examples accurate.
4. Match the documented endpoints to implemented behavior.
5. Verify generated docs after updates.

## Output

- Produce practical API documentation changes.
- Keep docs aligned with actual service behavior.
- Prefer accurate, minimal documentation over stale detail.
