# BullMQ Queues Web

Use this skill as a basic starting point.

## Workflow

1. Review the job processing flow and queue responsibilities.
2. Identify producer, worker, and retry behavior needs.
3. Keep payload contracts and job names explicit.
4. Handle failures, retries, and idempotency intentionally.
5. Verify queue behavior under normal and failure scenarios.

## Output

- Produce practical queue processing changes.
- Keep background workflows reliable and maintainable.
- Prefer explicit job boundaries over implicit behavior.
