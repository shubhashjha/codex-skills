# Mutation Testing .NET

Use this skill as a basic starting point.

## Workflow

1. Review the current test suite and critical business logic.
2. Identify where mutation testing adds the most value.
3. Keep the scope targeted to important logic paths.
4. Use results to improve test quality, not just coverage numbers.
5. Verify strengthened tests after findings are addressed.

## Output

- Produce practical test quality improvements.
- Keep verification focused on meaningful behavior.
- Prefer stronger assertions over broader but weaker tests.
