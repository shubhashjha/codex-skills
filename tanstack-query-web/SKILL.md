# TanStack Query Web

Use this skill as a basic starting point.

## Workflow

1. Review the current data fetching and state management pattern.
2. Identify query keys, mutation flows, and cache invalidation needs.
3. Keep query boundaries explicit and predictable.
4. Match loading, success, and error handling to the current UI.
5. Verify refetch and mutation behavior with realistic flows.

## Output

- Produce practical server state management changes.
- Keep data fetching maintainable and consistent.
- Prefer clear query contracts over scattered request logic.
