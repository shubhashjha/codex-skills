# Release Checklists

Use this skill as a basic starting point.

## Workflow

1. Review the type of release and operational impact.
2. Identify build, test, config, rollback, and communication steps.
3. Keep checklist items ordered and actionable.
4. Avoid vague items that cannot be verified.
5. Verify the checklist matches the deployment context.

## Output

- Produce practical release readiness guidance.
- Keep release steps explicit and reviewable.
- Prefer concise operational clarity over long generic lists.
