# Event Sourcing .NET

Use this skill as a basic starting point.

## Workflow

1. Review the domain model and event history requirements.
2. Identify aggregates, events, and replay boundaries.
3. Keep event contracts explicit and stable.
4. Handle versioning and read model implications carefully.
5. Verify state reconstruction from representative event streams.

## Output

- Produce practical event sourcing changes.
- Keep domain event design explicit and maintainable.
- Prefer clear event history over implicit state mutation.
