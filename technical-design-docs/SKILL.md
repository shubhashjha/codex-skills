# Technical Design Docs

Use this skill as a basic starting point.

## Workflow

1. Review the technical problem, constraints, and stakeholders.
2. Identify goals, non-goals, architecture choices, and tradeoffs.
3. Keep document structure direct and decision oriented.
4. Focus on implementation-relevant detail over generic theory.
5. Verify the design is understandable by the target audience.

## Output

- Produce practical design documentation.
- Keep decisions explicit and well-structured.
- Prefer clear tradeoff analysis over vague architectural language.
