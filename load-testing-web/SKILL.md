# Load Testing Web

Use this skill as a basic starting point.

## Workflow

1. Review the critical user flows and expected concurrency levels.
2. Identify realistic traffic patterns and bottleneck risks.
3. Keep scenarios measurable and representative.
4. Distinguish frontend rendering issues from backend capacity issues.
5. Verify results against baseline expectations.

## Output

- Produce practical load testing changes.
- Keep scenarios realistic and useful for decision making.
- Prefer representative traffic patterns over arbitrary stress.
