# Offline Sync Web

Use this skill as a basic starting point.

## Workflow

1. Review the offline requirements and current persistence model.
2. Identify sync boundaries, retry behavior, and conflict rules.
3. Keep local storage and server sync responsibilities explicit.
4. Surface offline status clearly to users.
5. Verify behavior across reconnect and conflict scenarios.

## Output

- Produce practical offline data handling changes.
- Keep sync behavior understandable and resilient.
- Prefer explicit conflict handling over hidden merge behavior.
