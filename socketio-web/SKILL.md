# Socket.IO Web

Use this skill as a basic starting point.

## Workflow

1. Review the current realtime requirements and event model.
2. Identify server, client, and room level responsibilities.
3. Keep event names and payloads explicit.
4. Handle reconnects, errors, and disconnects predictably.
5. Verify message flow across multiple clients.

## Output

- Produce practical realtime communication changes.
- Keep event flows understandable and maintainable.
- Prefer simple event contracts over ambiguous messaging.
