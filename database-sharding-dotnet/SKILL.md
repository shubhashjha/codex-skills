# Database Sharding .NET

Use this skill as a basic starting point.

## Workflow

1. Review current data volume, routing logic, and access patterns.
2. Identify shard keys, routing boundaries, and cross-shard risks.
3. Keep data access and shard resolution explicit.
4. Avoid broad changes that hide consistency tradeoffs.
5. Verify query and write flows against shard assumptions.

## Output

- Produce practical sharding oriented changes.
- Keep persistence logic explicit and supportable.
- Prefer clear shard rules over implicit routing.
