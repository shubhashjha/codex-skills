# Azure Container Apps

Use this skill as a basic starting point.

## Workflow

1. Review the containerized application structure and runtime requirements.
2. Identify ingress, environment, scaling, and secret needs.
3. Keep deployment configuration explicit and environment aware.
4. Avoid assumptions that belong to local Docker only.
5. Verify deployment behavior and container startup expectations.

## Output

- Produce practical container app deployment changes.
- Keep hosting configuration clear and supportable.
- Prefer explicit runtime settings over hidden defaults.
