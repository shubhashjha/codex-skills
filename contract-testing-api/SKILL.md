# Contract Testing API

Use this skill as a basic starting point.

## Workflow

1. Review the current API contracts and integration boundaries.
2. Identify producer and consumer expectations that must stay compatible.
3. Keep contracts explicit and version aware.
4. Avoid brittle tests tied to unrelated implementation details.
5. Verify both sides of the contract where possible.

## Output

- Produce practical contract testing changes.
- Keep service integration safer and more predictable.
- Prefer contract clarity over ambiguous integration assumptions.
