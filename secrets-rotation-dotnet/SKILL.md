# Secrets Rotation .NET

Use this skill as a basic starting point.

## Workflow

1. Review the current secret storage and configuration strategy.
2. Identify rotation frequency, fallback behavior, and dependency impact.
3. Keep rotation steps explicit and operationally safe.
4. Preserve compatibility during secret rollover windows.
5. Verify application behavior before and after rotation.

## Output

- Produce practical secret rotation changes.
- Keep operational security maintainable and predictable.
- Prefer safe rollout paths over abrupt secret changes.
