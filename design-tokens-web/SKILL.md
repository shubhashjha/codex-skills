# Design Tokens Web

Use this skill as a basic starting point.

## Workflow

1. Review the current colors, spacing, typography, and component styles.
2. Identify values that should become shared tokens.
3. Keep token naming consistent and scalable.
4. Minimize one-off values in component styles.
5. Verify token usage across multiple UI areas.

## Output

- Produce practical design token changes.
- Keep styling systems more reusable and consistent.
- Prefer shared primitives over duplicated values.
