# Theme Switching Web

Use this skill as a basic starting point.

## Workflow

1. Review the current color system and styling approach.
2. Identify theme tokens, persistence rules, and defaults.
3. Keep theme switching consistent across components.
4. Preserve readability, contrast, and accessibility.
5. Verify behavior across reloads and devices.

## Output

- Produce practical theme switching changes.
- Keep the implementation predictable and maintainable.
- Prefer token based theming over scattered overrides.
