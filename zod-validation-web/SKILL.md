# Zod Validation Web

Use this skill as a basic starting point.

## Workflow

1. Review the current data shape and validation boundaries.
2. Identify shared schemas between UI and API where relevant.
3. Keep schemas readable and close to domain requirements.
4. Return clear validation errors to the user or caller.
5. Verify valid and invalid input cases.

## Output

- Produce practical schema validation changes.
- Keep validation explicit and maintainable.
- Prefer reusable schemas over scattered checks.
