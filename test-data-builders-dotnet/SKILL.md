# Test Data Builders .NET

Use this skill as a basic starting point.

## Workflow

1. Review the current tests for repetitive setup noise.
2. Identify objects and scenarios that benefit from builders.
3. Keep builders focused, readable, and easy to override.
4. Avoid hiding important test intent behind opaque helpers.
5. Verify resulting tests are clearer than before.

## Output

- Produce practical test setup improvements.
- Keep tests easier to read and maintain.
- Prefer explicit builder defaults with simple overrides.
