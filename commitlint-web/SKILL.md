# Commitlint Web

Use this skill as a basic starting point.

## Workflow

1. Review the repository's current commit standards.
2. Identify the desired commit format and enforcement points.
3. Keep rules understandable and not overly restrictive.
4. Align tooling with the team's actual workflow.
5. Verify commit validation locally after changes.

## Output

- Produce practical commit convention changes.
- Keep workflow enforcement clear and maintainable.
- Prefer simple standards that teams will actually follow.
