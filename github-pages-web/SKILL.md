# GitHub Pages Web

Use this skill as a basic starting point.

## Workflow

1. Review the static site structure and deployment requirements.
2. Identify the correct publishing root and asset paths.
3. Keep the site compatible with static hosting constraints.
4. Match the repository structure to GitHub Pages expectations.
5. Verify deployed paths, assets, and downloads.

## Output

- Produce practical static deployment changes.
- Keep hosting setup simple and reliable.
- Prefer repo friendly deployment patterns.
