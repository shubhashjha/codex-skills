# Docker Compose Web

Use this skill as a basic starting point.

## Workflow

1. Review the app services, dependencies, and local environment needs.
2. Identify service boundaries, ports, volumes, and environment variables.
3. Keep compose definitions small and readable.
4. Avoid production assumptions in local-only orchestration.
5. Verify startup and service connectivity locally.

## Output

- Produce practical local orchestration changes.
- Keep multi-service setup understandable and maintainable.
- Prefer simple local environments over overbuilt compose files.
