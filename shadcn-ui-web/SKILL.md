# shadcn UI Web

Use this skill as a basic starting point.

## Workflow

1. Review the current component setup and styling system.
2. Identify existing shadcn/ui conventions in the project.
3. Reuse generated components instead of reinventing patterns.
4. Keep variants and composition simple and consistent.
5. Verify styling, accessibility, and interaction states.

## Output

- Produce practical shadcn/ui based changes.
- Keep components consistent with the current design system.
- Prefer reuse and composition over duplication.
