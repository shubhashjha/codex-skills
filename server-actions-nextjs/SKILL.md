# Server Actions Next.js

Use this skill as a basic starting point.

## Workflow

1. Review the current Next.js data fetching and mutation model.
2. Identify actions that fit server side execution.
3. Keep action inputs, outputs, and validation explicit.
4. Avoid mixing concerns between client and server boundaries.
5. Verify action behavior and error handling.

## Output

- Produce practical server action changes.
- Keep server and client concerns clear.
- Prefer simple mutation paths over unnecessary indirection.
