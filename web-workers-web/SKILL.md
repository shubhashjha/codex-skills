# Web Workers Web

Use this skill as a basic starting point.

## Workflow

1. Review the performance bottleneck and compute workload.
2. Identify tasks safe to move off the main thread.
3. Keep message passing and data contracts explicit.
4. Avoid overcomplicating simple synchronous work.
5. Verify responsiveness and worker lifecycle behavior.

## Output

- Produce practical background processing improvements.
- Keep worker boundaries clear and maintainable.
- Prefer measurable responsiveness gains.
