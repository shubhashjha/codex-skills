# React Error Boundaries

Use this skill as a basic starting point.

## Workflow

1. Review the current component tree and failure points.
2. Identify where boundary placement improves resilience.
3. Keep fallback UI useful and non-disruptive.
4. Preserve debugging visibility and logging where available.
5. Verify failure behavior in realistic scenarios.

## Output

- Produce practical error boundary changes.
- Keep failure handling user friendly and maintainable.
- Prefer targeted boundaries over blanket wrappers.
