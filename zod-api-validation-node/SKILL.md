# Zod API Validation Node

Use this skill as a basic starting point.

## Workflow

1. Review the request boundaries and data contracts.
2. Identify inputs that need schema validation.
3. Keep schemas close to the API layer where possible.
4. Return consistent validation errors to callers.
5. Verify valid and invalid request flows.

## Output

- Produce practical request validation changes.
- Keep input handling explicit and maintainable.
- Prefer shared schemas over scattered checks.
