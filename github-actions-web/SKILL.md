# GitHub Actions Web

Use this skill as a basic starting point.

## Workflow

1. Review the repository structure, scripts, and deployment expectations.
2. Identify the smallest useful CI or deployment workflow.
3. Keep workflow steps explicit and easy to debug.
4. Reuse caching and build steps only where they clearly help.
5. Verify workflow syntax and expected triggers.

## Output

- Produce practical CI workflow changes.
- Keep automation maintainable and repository friendly.
- Prefer clear build and deploy steps over overengineered pipelines.
