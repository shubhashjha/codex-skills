# Monorepo Turborepo

Use this skill as a basic starting point.

## Workflow

1. Review the repository layout, apps, and packages.
2. Identify shared dependencies and build task boundaries.
3. Keep scripts, caching, and pipeline configuration simple.
4. Avoid changes that break isolated package workflows.
5. Verify tasks at the app and workspace level.

## Output

- Produce practical monorepo workflow changes.
- Keep repository structure predictable and efficient.
- Prefer clear task boundaries over fragile automation.
