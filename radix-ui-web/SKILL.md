# Radix UI Web

Use this skill as a basic starting point.

## Workflow

1. Review the existing component composition and accessibility requirements.
2. Identify where Radix primitives fit the current interaction model.
3. Keep behaviors accessible and predictable.
4. Pair primitives with the existing styling approach.
5. Verify keyboard, focus, and screen reader behavior.

## Output

- Produce practical Radix UI based changes.
- Keep interactive components accessible and maintainable.
- Prefer composable primitives over custom low-level behavior.
