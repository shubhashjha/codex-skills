# Micro Frontend Web

Use this skill as a basic starting point.

## Workflow

1. Review the application boundaries and integration needs.
2. Identify ownership, deployment, and shared dependency constraints.
3. Keep interfaces between frontend modules explicit.
4. Minimize coupling and duplicated global state.
5. Verify routing, styling, and shared runtime behavior.

## Output

- Produce practical micro frontend changes.
- Keep module boundaries clear and supportable.
- Prefer simple composition over unnecessary fragmentation.
