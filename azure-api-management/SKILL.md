# Azure API Management

Use this skill as a basic starting point.

## Workflow

1. Review the current API surface and gateway responsibilities.
2. Identify authentication, routing, throttling, or transformation needs.
3. Keep policies explicit and environment aware.
4. Avoid duplicating backend business logic at the gateway.
5. Verify policy behavior on representative requests.

## Output

- Produce practical API gateway configuration changes.
- Keep gateway behavior controlled and maintainable.
- Prefer focused policies over complex gateway logic.
