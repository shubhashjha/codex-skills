# Performance Testing API

Use this skill as a basic starting point.

## Workflow

1. Review the critical endpoints and expected load behavior.
2. Identify latency, throughput, and bottleneck questions to answer.
3. Keep scenarios realistic and measurable.
4. Focus on endpoints that matter to users or operations.
5. Verify results against a clear baseline.

## Output

- Produce practical API performance testing changes.
- Keep tests measurable and actionable.
- Prefer realistic load scenarios over synthetic extremes.
