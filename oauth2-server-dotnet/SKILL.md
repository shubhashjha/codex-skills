# OAuth2 Server .NET

Use this skill as a basic starting point.

## Workflow

1. Review the current identity and token flow architecture.
2. Identify grant types, client types, and token validation requirements.
3. Keep scopes, claims, and lifetimes explicit.
4. Avoid weakening token security for convenience.
5. Verify authentication and authorization flows end to end.

## Output

- Produce practical OAuth2 server changes.
- Keep token handling secure and maintainable.
- Prefer explicit flow design over ad hoc auth behavior.
