# Frontend Observability Web

Use this skill as a basic starting point.

## Workflow

1. Review the current logging, monitoring, and error reporting setup.
2. Identify important events, failures, and performance signals to capture.
3. Keep instrumentation useful and not overly noisy.
4. Respect privacy and sensitive data boundaries.
5. Verify captured signals during realistic flows.

## Output

- Produce practical frontend observability changes.
- Keep monitoring actionable and maintainable.
- Prefer high-signal instrumentation over broad noise.
