# Outbox Pattern .NET

Use this skill as a basic starting point.

## Workflow

1. Review the current transaction and messaging boundaries.
2. Identify where state changes and event publication must stay consistent.
3. Keep outbox records, processing, and retries explicit.
4. Preserve idempotency and failure visibility.
5. Verify message flow under success and retry scenarios.

## Output

- Produce practical outbox pattern changes.
- Keep reliability mechanisms maintainable and explicit.
- Prefer transactional clarity over hidden event publishing.
